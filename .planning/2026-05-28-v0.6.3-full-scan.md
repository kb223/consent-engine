## Decision
Production (`run_audit`) currently calls `scan_page_fast`, which only injects
OneTrust's cookies (no `build_injection_plan`, no banner-click), so every
non-OneTrust CMP comes back INCONCLUSIVE. The full-scan family
(`scan_page` -> `_scan_s1`/`_scan_s3`/`_scan_gpc`) has multi-CMP injection +
banner-click + GPC variant + a 150s timeout wrapper, but lacks the newer wiring
(CMP runtime introspection, consent-event capture, stealthy WAF retry).

**Promote the full scan; port the 3 newer features into it.** (Porting ~3 light
features is lower-risk than porting the heavy banner-click logic into the fork.)
