## v0.6.4 — per-CMP reject-click reliability (Sourcepoint) — 2026-05-28

Systematic-debugging pass on cmp_clicker.py. Live evidence captured under
/tmp/ce-clicker-diag/ (LOCAL ONLY, never committed). Reproduced the engine's
exact S3 path (same stealth config, same detect_cmp + attempt_cmp_decline).

### Root causes found (4 compounding bugs on Sourcepoint)
1. **Wrong JS API.** Clicker called `window._sp_.rejectAll(campaignType)` — does
   NOT exist on modern builds (verified live: `_sp_.rejectAll` undefined). Real
   API is namespaced: `_sp_.usnat.postRejectAll()` (US, confirmed on bbc),
   `_sp_.ccpa.rejectAll()` (legacy); GDPR exposes only PM loaders. -> fixed to
   try real methods; GDPR falls through to the iframe click.
2. **Iframe never entered.** Guardian's reject ("No, thank you") lives in
   `iframe[id^='sp_message_iframe']` (CNAME sourcepoint.theguardian.com). The
   clicker only entered frames matching `"truste"`, AND gated entry on
   `dom_type=="iframe"` — but Sourcepoint reports `dom_type=="standard"`. ->
   new `_select_cmp_frame()` enters any CMP message iframe regardless of dom_type.
3. **Vocab gap.** Reject label "No, thank you" wasn't in REJECT_VOCAB. -> added
   "no, thank you", "do not consent", "continue without accepting".
4. **`_banner_present` false-positive (THE blocker).** After a *successful*
   reject (SP iframe gone), `_banner_present` still returned True by matching
   `[class*='banner']` on a non-consent element — Guardian's subscription promo
   `<span class="rr_banner_highlight">`. So the loop reported a good reject as
   `banner_click_failed`. -> `_BANNER_SELECTORS` is now consent-specific (explicit
   CMP containers + sp_message_iframe + cookie/consent keyword selectors); bare
   `[class*='banner']` removed. Affects ALL CMPs (accuracy win across the board).

### KEY FINDING — guardian/bbc/cnn emit ZERO GCS
The named publisher sites do NOT use Google Consent Mode (0 `gcs=` params across
255 / 181 / 245 requests). They're IAB-TCF / US-privacy publishers. The engine's
verdict is GCS-based, so a perfect reject can't make them "definitive" — INCONCLUSIVE
is now honest (no signal exists), not a false click failure. The clicker fix WILL
make GCS-*using* Sourcepoint sites definitive (now that reject + GCS-observe works).
